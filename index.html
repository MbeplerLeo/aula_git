Guia Git & GitHub

O que é o Git

O Git é um sistema de controle de versão distribuído e gratuito, criado por Linus Torvalds. Ele permite que desenvolvedores versionem, testem e colaborem em código de forma eficiente e segura.

O que é o GitHub

O GitHub é uma plataforma que hospeda repositórios Git na nuvem, permitindo colaboração, controle de versão, integração com CI/CD, pull requests e muito mais.

Diferenças entre Git e GitHub

  • Git: ferramenta de versionamento local.
  • GitHub: serviço online para armazenar e gerenciar repositórios Git.

Principais comandos Linux para usar no Git Bash

  • ls – Lista arquivos e pastas
  • cd – Acessa diretórios
  • mkdir – Cria diretórios
  • touch – Cria arquivos
  • clear – Limpa o terminal
  • rm – Remove arquivos

Comandos Linux do repositório aula_git

  • mkdir aula_git
  • cd aula_git
  • touch index.html
  • code .

Como criar um repositório no GitHub

  1. Entre no GitHub
  2. Clique em New repository
  3. Dê um nome, configure visibilidade, adicione README (opcional)
  4. Clique em Create repository

Como ligar em um projeto existente na máquina

  1. Abra o terminal na pasta do projeto
  2. git init
  3. git remote add origin URL_DO_REPOSITORIO

Como gerar uma versão com Git

  1. git add .
  2. git commit -m "Mensagem do commit"

Como enviar da máquina para o GitHub

  1. git push -u origin main
